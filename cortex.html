Agent Detaylari

REVIEWER — Kod Incelemeci

  • Ne yapar: Bitbucket'ta yeni bir PR acildiginda otomatik tetiklenir. Kod kalitesini, guvenlik acikllarini ve best practice uyumunu analiz eder.
  • Cikti: PR uzerinde yorum olarak inceleme raporu birakir.
  • Sinirlamalar: Merge yapamaz, sadece yorum birakir.

TESTER — Test Yazici

  • Ne yapar: Verilen kaynak kod icin otomatik unit test yazar.
  • Cikti: Test dosyalari + PR olusturur.
  • Onay: Commit oncesi insan onayi gerektirir.

DEVELOPER — Gelistirici

  • Ne yapar: Jira'da P2-P4 oncelikli bug'lar icin otomatik kod duzeltmesi yapar. P1 bug'lara DOKUNAMAZ.
  • Cikti: Bug fix kodu + PR olusturur.
  • Onay: PR olusturma oncesi insan onayi zorunludur.
  • Kritik: Yeni ozellik (feature) YAZAMAZ, sadece bug fix yapar.

PLANNER — Planlamaci

  • Ne yapar: Karmasik gorevleri alt gorevlere parcalar, uygulama plani olusturur.
  • Cikti: Yapilandirilmis plan dokumani (adimlar, bagimliliklar, tahmini efor).
  • Kullanim: E2E pipeline'da ilk adim olarak calisir.

INDEXER — Indeksleyici

  • Ne yapar: Kaynak kodu, Jira issue'lari ve Confluence sayfalarini vektor veritabanina (Qdrant) indexler.
  • Cikti: Qdrant'ta aranabilir vektor verileri.
  • Neden onemli: Diger agent'larin semantik arama yapmasi icin veri hazirlar.

GAP_ANALYZER — Bosluk Analizcisi

  • Ne yapar: Dokumantasyon ile gercek kod arasindaki uyumsuzluklari tespit eder.
  • Cikti: Uyumsuzluk raporu (hangi dokuman neyle celiyor, eksik/fazla ne var).
  • Ongirt: Qdrant'ta indexlenmis veri olmasi gerekir (INDEXER calismis olmali).

MFE_DEVELOPER — Frontend Gelistirici

  • Ne yapar: React bilesenlerini ve Storybook hikayelerini otomatik uretir.
  • Cikti: React component + Storybook story dosyalari + PR.
  • Onay: PR olusturma oncesi insan onayi zorunludur.

REQUIREMENTS_ANALYST — Gereksinim Analisti

  • Ne yapar: Confluence sayfalarindan PBI (Product Backlog Item) ve test case'leri otomatik cikarir.
  • Cikti: Yapilandirilmis gereksinim listesi ve test senaryolari.
  • Kullanim: Analistler icin en faydali agent — mevcut dokumantasyonu yapisal hale getirir.

3.3 Agent Guvenlik Kurallari

  • P1 bug'lar ENGELLENDI — Sadece insanlar cozebilir, agent asla P1'e dokunamaz
  • Yeni ozellik ENGELLENDI — Agent sadece bug fix yapabilir, yeni feature yazamaz
  • Merge YAPAMAZ — Agent sadece PR acar, merge karari insana aittir
  • Maliyet takibi — Her agent calistirmasinda token kullanimi ve maliyet kaydedilir.
